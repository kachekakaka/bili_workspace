# bili_workspace V0.5.4 发布与验证说明

> 本文件是 V0.5.4 的历史发布记录，不代表当前 `main` 的目录结构和全部功能。当前文档入口见 `docs/README.md`；当前部署使用 `config`、`userdata`、`downloads` 三类持久化边界。

## 发布目的

V0.5.4 将项目恢复为标准、可直接克隆和审查的 Git 仓库，并固化当时的 Windows、QNAP Docker、配置升级、局域网/域名访问和设备导出边界。

## 当时的关键变化

- 移除 bootstrap、Base64 源码分片和 Actions 自恢复方案；
- `main` 直接保存完整 Python、前端、测试、Docker 和 Windows 脚本；
- 所有可变配置采用 `.default` 模板，实际配置保持未跟踪；
- 缺失 JSON 字段递归补入，dotenv 缺失键增量追加，不覆盖已有值；
- 支持可配置 IP/主机名和 1–65535 端口，非回环监听强制认证；
- Windows `setup.bat` 可从固定运行资产补齐 BBDown、FFmpeg 和依赖并校验哈希；
- 修复配置文件使用局域网主机名时可信 Host 未自动同步的问题，并加强主机名语法校验；
- 保留媒体库、分组、清晰度控制、浏览器播放、一次性设备导出和网页扫码登录；
- 增加长期产品需求与架构基线，避免需求仅保存在聊天上下文。

## 当前主线与 V0.5.4 的差异

当前 `main` 已继续变化：

```text
Windows：config/、userdata/、downloads/
Docker：/data/config、/data/userdata、/downloads
```

并新增搜索词模式、标签和无标签筛选、作品分组修改、任务原 ID 重试与批量控制、作品库多字段排序，以及独立的删除历史记录。

因此旧的 config/media/cache/tmp 四目录描述、发布时测试数量和当时运行资产获取方式只用于历史追溯。

## Windows 完整包验证边界（历史）

当时完整 Windows 恢复包额外进行：

- BBDown 和 FFmpeg 固定 SHA-256 校验；
- 发布清单逐文件校验；
- Cookie、旧虚拟环境、缓存、ffplay、ffprobe 和数据库扫描；
- ZIP CRC、路径穿越和符号链接检查。

当前 Windows 已使用仓库内 `vendor/windows/*.pack` 和 `runtime-manifest.json`，实际使用应以当前主线文档为准。

## QNAP / Docker 验证边界

实际 QNAP Container Station 构建、PUID/PGID、反向代理来源 IP、证书和 NAS 硬件性能始终需要在目标 QNAP 上验收。当前部署步骤见《QNAP Docker 部署指南》。
