# V0.6.2 UI 发布与验证

## 版本定位

V0.6.2 是 V0.6.1 之上的 UI/UX 修正版，不增加新的下载业务能力，也不修改数据库 schema、API 协议或权限模型。

## 用户可见改动

- 桌面端 Button/Input/Select 普通高度统一为 40px，紧凑控件为 32px；
- 手机和平板可触控控件保持不低于 44px；
- 创建下载页输入区域、页面顶部和卡片留白适度压缩；
- 用户显示名、临时密码和分组重命名统一使用站内 Modal，不再调用浏览器 prompt；
- 账号页拆成“Bilibili 登录”和“网站账号与设备”两个页签；
- 网站账号页显示用户名、中文显示名、修改密码和最多 10 个有效 Token/登录设备；
- 用户、分组、任务拥有者等动态选项超过 8 项时，使用带搜索框的两列选择列表；
- 清晰度、编码等固定少量选项继续使用普通 Select；
- 设置页默认展示常用配置，任务超时、轮询、画质和编码策略进入高级设置折叠区。

## 实现边界

本版本不修改：

- Bilibili 搜索和下载算法；
- 任务所有权、历史保留和设备导出规则；
- SQLite schema v4；
- 会话 Token 上限和淘汰算法；
- 后端 API 路由与权限矩阵；
- 标签系统和作品库数据模型。

覆盖：

- 单一浏览器资源缓存批次；
- 32/40/48px 控件 Token 和移动端 44px 触控下限；
- 原生 prompt 不被调用；
- 用户显示名、临时密码和分组重命名 Modal；
- Bilibili 与网站账号页签分离；
- 登录设备列出、单设备撤销和退出其他设备；
- 动态选择项搜索和赋值；
- 高级设置折叠后不参与页面布局；
- 1920×1080、1440×900、1024×768、768×1024、390×844 五档视口无横向溢出或可见控件重叠。

## CI 门禁

PR 合并前必须全部通过：

- Python 3.11、3.12、3.13；
- Ruff；
- 完整 pytest；
- 全部 JavaScript `node --check`；
- Markdown 内部链接；
- V0.6.2 UI 专项 Playwright；
- 原有五档固定视口 Playwright；
- Windows 便携运行时与 `verify.bat`；
- Docker 构建、schema v4 迁移和镜像内版本冒烟。

## 更新与回滚

本版本没有数据库迁移。更新仍使用：

```bash
git pull --ff-only origin main
```

Windows 运行 `start.bat`，需要完整自检时运行 `verify.bat`；Docker/QNAP 继续使用原来的 `config`、`userdata` 和 `downloads` 三目录映射。

回滚代码前仍应停止应用并备份当前 `userdata`。由于 schema 仍为 v4，可以回到已验证的 V0.6.1 提交，但不得忽略运行期间产生的新任务和会话数据。
