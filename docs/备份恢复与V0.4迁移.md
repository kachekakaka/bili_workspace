# 备份、恢复与旧版本迁移

## Docker 需要备份

```text
CONFIG_DIR  配置、SQLite、管理员、任务、分组、观看进度和 Bilibili 会话
MEDIA_DIR   永久媒体文件与 .bili_index.json
```

可选：

```text
CACHE_DIR   封面和兼容副本，可重建
```

无需备份：

```text
TEMP_DIR    下载、转码和一次性设备导出临时文件
```

## Windows 需要备份

```text
config/                         配置、SQLite 和服务器状态
downloads/                      媒体与下载索引
BBDown_portable/BBDown.data     Bilibili 会话，必须加密和限制访问
```

`.env` 和 `config/runtime.env` 包含网络/域名设置，也应随配置目录或安全配置备份。

无需备份：

```text
.venv/
.cache/
.tmp/
.pytest_cache/
__pycache__/
```

## 一致性备份

推荐先停止服务或容器。在线备份 SQLite 时必须同时保留主数据库、`-wal` 和 `-shm` 文件，不能只复制主 `.db`。

## Docker 恢复

1. 停止容器；
2. 恢复原 `CONFIG_DIR` 与 `MEDIA_DIR`；
3. 确认 PUID/PGID 可读写；
4. 运行 `./docker/build-and-start.sh`；
5. 检查管理员、作品数量、分组、任务和播放记录。

## Windows 恢复

1. 停止网站；
2. 恢复 `config/`、`downloads/` 和需要的 `BBDown.data`；
3. 从 GitHub 更新源码；
4. 运行 `setup.bat` 和 `verify.bat`；
5. 启动并核对数据。

## 从旧 Windows 版本迁移

- 旧根目录 `config.json`：当 `config/config.json` 不存在时，配置同步会先复制旧文件，再补充新版缺失字段；
- 旧 `downloads/`：整体保留或复制到新目录，V0.5 会读取 `.bili_index.json` 并增量导入 SQLite；
- 不复制旧 `.venv`，应在新目录重新创建；
- 已登录的 `BBDown.data` 可保留，但不要提交 Git 或通过聊天传输。

## 从 V0.4 Windows 迁移到 QNAP

1. 停止旧版；
2. 将旧 `downloads` 内容复制到 Docker 的 `MEDIA_DIR` 根目录，包括 `.bili_index.json`、`groups/` 和旧 `items/`；
3. 配置并启动容器；
4. 核对作品数量、路径和文件大小；
5. 确认无误后再决定是否删除 Windows 副本。

## 索引与数据库

- `.bili_index.json` 保存下载产物和安全删除所需文件路径；
- SQLite 负责作品库分页、分组显示、观看进度、管理员、任务和导出状态；
- 分组重命名默认只更新逻辑归属，不大规模移动媒体文件；
- 不要手工写入任意路径，不安全路径会被拒绝。
