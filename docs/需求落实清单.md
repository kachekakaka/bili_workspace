# 当前主线需求落实清单

本清单对应当前 `main`。当前发布版本为 `0.7.0`，V0.7.0 前端结构整理 PR 1–8 已按冻结方案完成。

V0.6.2 UI/UX 修正版继续作为控件高度、Modal、可搜索 Select、账号页签和固定视口的兼容验收基线。

## 搜索

- 单一 `/api/search` 与单一正式 Search renderer；
- 每次只请求 Bilibili 当前页，每页 20 条；
- 精准/模糊只对当前页标题做全部词/任意词匹配，不联网；
- 成功后最多预加载下一页 1 页，不抓取第三页或全部页面；
- 修改关键词/排序和切换路由会取消旧请求，旧响应不能覆盖新页面；
- 支持缓存、已下载/已删除识别、原作品页、跨页选择、标签、画质预览和批量下载；
- 创建任务后保留 Search route 和当前页。

## 下载、任务和媒体库

- 媒体库与当前设备导出两种目标；普通用户被服务端强制为设备导出；
- 最低清晰度与指定档位在下载前后核验；
- 任务显示阶段、进度、大小、速度、ETA、分 P 和日志；
- 单项/批量暂停、继续、取消、重试、画质编辑和删除；
- 重试复用原任务 ID；
- Library 支持分页、关键词、分组、标签、无标签、编码、清晰度、观看状态和多字段升降序；
- 支持播放器、Range、拖动、续播、多文件/分 P、改分组和批量标签/下载/删除；
- 删除历史使用独立 `deleted_media`，显式重新下载成功后清除。

## 多用户前端、会话和权限

- 一个管理员和多个普通用户；
- 每次登录使用独立 HttpOnly Token，SQLite 只保存哈希；
- 每用户最多 10 个有效 Token，第 11 个淘汰最久未连接设备；
- 首次强制改密、Token/CSRF 轮换、本人设备管理和管理员用户管理；
- 任务、日志、SSE 和导出按 `owner_user_id` 隔离；
- 普通用户只显示 Download/Tasks，最多 10 个活动任务，终态保留 7 天且最多 100 条；
- 管理员可查看全部用户任务并筛选、排序和分组。

## V0.7.0 前端结构

- 原生 `.mjs` 和静态 import，无 npm、打包器或转译器；
- `index.html` 只有一个应用入口；
- 每个页面一个 renderer，统一 mount/dispose；
- Router generation + AbortController 防止旧响应写入；
- API、Session、Context、Modal、Toast、Confirm、TaskStream 各一份；
- Dashboard/Tasks 共享一个 EventSource；
- CSS 为 tokens/base/components/pages；
- 无旧 `app.js`、enhancement/overlay、Legacy Bridge、页面 MutationObserver、捕获阶段抢占或原生 prompt/confirm；
- Groups/Users/Tasks/Library 跨页面交接使用一次性路由状态，不使用全局业务状态；
- 浏览器与服务版本不一致时提供明确恢复提示。

## 配置、数据库和持久化

- Git 只提交 `.default` 模板，不提交实际配置；
- Windows：`config/`、`userdata/`、`downloads/`；
- Docker：`/data/config`、`/data/userdata`、`/downloads`；
- SQLite、任务、索引、日志、缓存和临时状态位于 `userdata`；
- schema 继续为 v4，V0.7.0 不新增数据库迁移；
- 更新源码和重建容器不会重置配置、数据库或媒体。

## 性能与可靠性

- 搜索有 WBI 和原始页缓存，最多预载下一页；
- TaskStream 单连接、多订阅者、登出关闭；
- Library 服务端筛选、排序和分页；
- 大文件分块/Range，客户端断开停止读取；
- 下载和转码并发受限并检查剩余空间；
- 日志、导出、缓存和历史记录有容量或 TTL 边界。
