# 当前主线需求落实清单

本清单对应当前 `main`，用于快速确认已经落地的用户需求。具体架构约束见《产品需求与架构基线》。

> V0.6.0 的 PR 1–4 已经进入当前主线；版本升级和最终发布验收仍未完成。完整开发目标见 [`plans/V0.6.0_多用户搜索与会话方案.md`](plans/V0.6.0_多用户搜索与会话方案.md)，本文件只记录已经合并的能力。

## 搜索

- 搜索 API 和前端均已收敛为单一路由、单一状态和单一正式 renderer；
- 点击搜索只请求 Bilibili 原始当前页，每页 20 条；
- 不筛选、精准和模糊是当前页标题二级筛选，切换模式和修改筛选词均不发起网络请求；
- 精准模式要求标题包含全部拆分词，模糊模式要求标题包含任意拆分词；
- UP 主和 BV 号不参与标题二级匹配；
- 当前页成功渲染后最多预加载下一页 1 页，不预加载第三页，也不会后台抓取十页；
- 修改 B站关键词或排序会取消旧请求和旧预加载，旧响应不能覆盖新结果；
- WBI 密钥缓存 10 分钟，原始搜索页缓存 3 分钟；“刷新B站结果”只刷新当前页；
- 搜索响应直接包含标签、下载状态和删除状态，不再额外批量请求标签；
- 默认“屏蔽已下载和已删除”，关闭后可辨识并确认重新下载；
- 支持原作品页直达、跨页选择、批量标签、画质预览和批量下载；
- 创建下载任务后保留当前页面；
- 搜索与当前管理员页面已通过 1920×1080、1440×900、1024×768、768×1024、390×844 固定视口检查，无横向滚动或控件重叠。

## 下载、任务和清晰度

- 下载时可选择已有分组并立即新建分组；
- 下载前预览最高可用清晰度、分辨率、编码、帧率和估算大小；
- 最低清晰度与指定档位进行下载前预检和下载后实际码流核对；
- 任务中心显示标题、BV、阶段、时长、分 P、当前大小、总大小、速度、ETA 和日志；
- 支持单任务和批量暂停、继续、取消、重试、删除；
- 支持一键重试失败任务和一键清理失败/取消记录；
- 失败任务可以修改清晰度后重新下载；
- 重试复用原任务 ID，不创建重复任务；
- 搜索页和创建下载页提交后不自动跳转任务中心。

## 多用户任务、日志与导出

- 每个任务都以认证上下文中的 `owner_user_id` 为唯一拥有者，客户端提交的拥有者字段不会改变归属；
- 普通用户只能创建设备导出任务，服务端无条件覆盖媒体库目标、分组和强制替换字段；
- 普通用户只能查看和操作自己的任务、日志和导出文件；越权访问统一按不存在返回；
- 普通用户 SSE 只推送自己的任务，管理员可查看全部任务并按用户、状态、目标、关键词和时间字段筛选排序；
- 不同用户可以同时导出同一个 BV，临时目录和导出索引按用户内部命名空间隔离；
- 普通用户最多 10 个排队或运行中的活动任务，第 11 个被拒绝；终态任务保留 7 天且最多 100 条；
- 管理员终态任务最多保留 500 条，不受普通用户 7 天规则影响；活动任务和尚未完成传输的导出不会被历史清理；
- schema v4 将既有任务与导出归属原管理员，v3 升级不会再次撤销 PR 2 已建立的有效会话。

## 多用户前端

- 普通用户桌面侧边栏和手机底栏只显示“下载”和“任务”，访问其他 hash 会回到下载页；
- 普通用户下载页只显示作品链接/编号、最低清晰度和“导出到当前设备”提示，不显示目标、分组、标签、force 或媒体库路径；
- 普通用户任务页只显示本人任务，提供日志、暂停、继续、取消、重试、删除和设备导出下载；
- 顶部用户菜单显示中文显示名、登录账号和角色，并提供修改显示名、修改密码、登录设备和退出登录；
- 管理员用户管理页支持创建普通用户、修改显示名、禁用/启用、设置临时密码、撤销会话和跳转该用户任务；
- 管理员用户管理页在桌面使用表格，在手机使用卡片；任务中心可按用户筛选、排序和分组显示，任务卡明确显示 `显示名（username）`；
- 登录、强制改密、管理员页面和普通用户双页面均纳入 1920×1080、1440×900、1024×768、768×1024、390×844 Playwright 检查。

## 作品库、分组和标签

- SQLite 作品库提供服务端分页、筛选和排序；
- 支持按下载时间、最近观看、标题、时长、大小、分组和标签升降序排列；
- 作品封面完整显示，不强制裁切主体；
- 支持播放、Range、拖动、续播、多文件和多分 P 切换；
- 分组管理支持浏览、重命名、合并和删除空分组；
- 作品卡片和播放详情均可直接修改分组；
- 搜索结果和作品库均可添加标签；
- 分组和标签筛选使用横向 chip 条；
- 标签筛选包含“无标签”；
- 默认标签使用不同颜色，用户自定义颜色优先；
- 筛选后的作品支持跨页批量标签、下载和删除。

## 删除历史

- 删除作品会从作品库移除，并按选择删除媒体文件；
- 删除历史写入 SQLite 独立 `deleted_media` 表；
- 删除历史不是普通标签，不会作为作品重新显示；
- 搜索可以识别并显示“已删除”；
- 默认搜索屏蔽同时隐藏已下载和已删除作品；
- 用户确认重新下载成功后，对应删除记录清除。

## 播放与导出到当前设备

- 手机、平板和桌面浏览器均可在线播放；
- 媒体接口支持 HEAD、Range、206、416、ETag 和续播；
- 默认优先播放原文件，不兼容时手动生成 H.264/AAC MP4 副本；
- 下载目标可选媒体库或当前浏览器设备；
- 设备导出完成发送后立即删除服务器临时文件；
- 中断传输不删除，允许重试，并由 TTL 清理兜底；
- 多文件或多分 P 可打包，流式发送避免把大文件整体载入内存。

## 网站账号、会话、网络和域名

- 所有运行模式均要求网站账号登录；Windows 回环全新安装创建 `admin / 123456` 临时管理员并强制首次改密；
- Docker/NAS 和非回环初始化继续使用一次性令牌，默认临时密码不能远程登录；
- 支持一个管理员和多个普通用户账号，登录账号、中文显示名和正式密码按固定规则校验；
- 每次登录创建独立 HttpOnly Cookie Token，SQLite 只保存 Token 哈希；
- 每用户最多 10 个有效 Token，第 11 次登录按 `last_seen_at`、`created_at` 淘汰最久未连接会话；
- 改密会轮换当前 Token 与 CSRF、撤销其他会话；用户可查看和撤销自己的其他设备；
- 管理员 API 可创建普通用户、修改显示名、禁用/启用、重置临时密码和撤销全部会话；
- 强制改密会话只能访问登录状态、改密和退出；普通用户当前不能访问管理员功能；
- Windows 默认回环监听，但地址与端口均可配置；支持 `0.0.0.0`、指定局域网 IP、IPv6、主机名和 1–65535 端口；
- 设置页可展示多网卡可访问地址和代理绕过提示；
- 支持 QNAP HTTPS 反向代理、可信 Host、可信代理、安全 Cookie、CSRF、HSTS 和登录限速；
- 网页可显示 Bilibili 二维码并完成扫码登录，完整 Cookie 不返回浏览器；
- 普通用户任务、日志、SSE 和导出已按拥有者隔离；双页面导航、精简下载页、用户菜单和管理员用户视图均已完成。

## 配置、数据库和 Docker 持久化

- Git 只提交 `.default` 模板，不提交实际配置；
- 标签模板为 `config/tags.json.default`，实际标签定义为 `config/tags.json`；
- 首次启动从模板创建，升级时只补缺失字段或 ENV 键，不覆盖用户值；
- Windows 使用 `config/`、`userdata/`、`downloads/` 三类目录；
- Docker 固定映射 `/data/config`、`/data/userdata`、`/downloads`；
- SQLite、任务、删除历史、索引和日志位于 `userdata`；schema 升级前在 `userdata/backups/` 创建一致性备份并只保留最近 3 份；
- 更新源码和重建容器不会重置配置、数据库或媒体；
- 容器非 root、只读根文件系统、丢弃 capabilities、不挂载 Docker socket。

## 性能与可靠性

- SSE 推送任务变化；
- 作品库服务端筛选、排序和分页；
- 搜索使用 10 分钟 WBI 密钥缓存和 3 分钟原始页缓存，浏览器最多预加载下一页 1 页；
- 封面缓存、大小限制和过期清理；
- 大文件分块传输，客户端断开后停止读取；
- 下载和转码并发受限，并在开始前检查剩余空间；
- 任务、标签、删除历史、分组和播放记录重启后保留；
- 日志、导出和缓存都有容量或 TTL 边界。
