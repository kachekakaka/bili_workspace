# V0.5.3 需求落实清单

## 下载、分组和清晰度

- 下载时可选择已有分组并立即新建分组；
- 分组管理支持浏览、重命名、合并、移动作品和删除空分组；
- 搜索、预览、任务和作品库同时显示标题与 BV 号；
- 下载前预览最高可用清晰度、分辨率、编码、帧率和估算大小；
- 最低清晰度与指定档位使用前后两次校验，禁止静默保存低清文件；
- 任务中心显示预检与实际清晰度、编码、速度、ETA、分段核对和完整日志。

## 播放与导出到当前设备

- 手机、平板和桌面浏览器均可在线播放；
- 媒体接口支持 HEAD、Range、206、416、ETag 和续播；
- 默认优先播放原文件，不兼容时手动生成 H.264/AAC MP4 副本；
- 下载目标可选 NAS 媒体库或当前浏览器设备；
- 设备导出完成发送后立即删除服务器临时文件；
- 中断传输不删除，允许重试，并由 TTL 清理兜底；
- 多文件或多分 P 自动打包，流式发送避免将大文件整体载入内存。

## 网络、认证和域名

- Windows 默认回环监听，但地址与端口均可配置；
- 支持 `0.0.0.0`、指定局域网 IP、IPv6、主机名和 1–65535 端口，包括 3389；
- 非回环监听自动切换服务器模式并强制管理员认证；
- 支持 QNAP HTTPS 反向代理、可信 Host、可信代理、安全 Cookie、CSRF、HSTS 和登录限速；
- 网页可显示 Bilibili 二维码并完成扫码登录，完整 Cookie 不返回浏览器。

## 配置、数据库和 Docker 持久化

- Git 只提交 `.default` 模板，不提交实际配置；
- 首次启动从模板创建，升级时只补缺失字段或 ENV 键，不覆盖用户值；
- SQLite、任务、分组、管理员、会话和观看进度位于配置目录；
- Docker 固定映射 `/data/config`、`/data/media`、`/data/cache`、`/data/tmp`；
- 更新源码和重建容器不会重置目录、配置、数据库或媒体；
- 容器非 root、只读根文件系统、丢弃 capabilities、不挂载 Docker socket。

## 性能与可靠性

- SSE 推送任务变化，轮询作为后备；
- 作品库服务端筛选、排序和分页；
- 封面缓存、大小限制和过期清理；
- 大文件分块传输，客户端断开后停止读取；
- 下载和转码并发受限，并在开始前检查剩余空间；
- 任务快照节流写入 SQLite，NAS 重启后可恢复队列状态；
- 日志、导出和缓存都有容量或 TTL 边界。
