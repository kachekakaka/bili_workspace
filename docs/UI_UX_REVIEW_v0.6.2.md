# UI/UX Review V0.6.2

## 状态

V0.6.2 范围已实现。版本不增加业务功能，重点优化前端交互和视觉一致性，并保持下载、数据库、API 与权限模型不变。

目标：从后台管理面板风格调整为家庭 NAS 媒体中心风格。

## 本版本范围

### P0（已完成）

- 统一 Button/Input/Select 高度和间距；
- 去除浏览器 prompt 类交互，统一使用 Modal；
- 分离网站账号管理和 Bilibili 账号管理。

### P1（已完成）

- 优化动态下拉选择体验；
- 设置页面区分常用设置和高级设置；
- 压缩过大的输入区域和筛选区域。

### P2（已完成）

- 优化页面留白；
- 降低卡片和列表的视觉高度；
- 微调桌面、平板和手机布局。

## 控件规范

桌面端默认：

- 普通控件：40px
- 紧凑控件：32px
- 主要操作按钮：48px

移动端可触控按钮、输入和选择控件不低于 44px。禁止同一页面出现高度明显不同的普通输入框、按钮和下拉框。

## Selector 规范

不引入复杂组件体系，只提供两类选择能力：

- 固定少量选项：普通 Select；
- 动态列表：支持搜索的 Select。

适用：

- 清晰度、编码等固定配置使用普通选择；
- 用户、分组、任务拥有者等动态数据超过 8 项时，使用带搜索框的两列选择列表；
- 动态数据不足 9 项时保留原生 Select，避免过度设计。

## 账号与会话规范

- “Bilibili 登录”只展示扫码、Cookie 和 Bilibili 登录状态；
- “网站账号与设备”展示用户名、中文显示名、密码和登录设备；
- 每用户最多 10 个有效 Token，用户可撤销单个其他设备或一次退出全部其他设备；
- 管理员修改显示名、设置临时密码和分组重命名全部进入 `modalRoot`。

## 设置层级

默认展开：

- 监听地址与端口；
- 媒体目录；
- 默认分组；
- 默认最低清晰度。

高级折叠：

- 任务超时；
- 页面轮询后备间隔；
- 画质优先级；
- 编码优先级。

折叠内容关闭时必须从布局中显式隐藏，不能与保存按钮重叠。

## 明确不做

- 不重构标签系统；
- 不增加首页大屏数据；
- 不重写下载流程；
- 不引入新的前端框架；
- 不修改数据库 schema、API 协议或权限模型。
