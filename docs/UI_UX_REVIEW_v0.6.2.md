# UI/UX Review V0.6.2

## 目标

V0.6.1 功能已经进入稳定阶段。V0.6.2 不增加业务功能，重点优化前端交互和视觉一致性。

目标：从后台管理面板风格调整为家庭 NAS 媒体中心风格。

## 本版本范围

### P0

- 统一 Button/Input/Select 高度和间距。
- 去除浏览器 prompt 类交互，统一使用 Modal。
- 分离网站账号管理和 Bilibili 账号管理。

### P1

- 优化下拉选择体验。
- 设置页面区分常用设置和高级设置。
- 压缩过大的输入区域和筛选区域。

### P2

- 优化页面留白。
- 降低作品卡片和列表视觉高度。
- 微调页面布局。

## 控件规范

桌面端默认：

- 普通控件：40px
- 紧凑控件：32px
- 主要操作按钮：48px

禁止同一页面出现高度明显不同的普通输入框、按钮、下拉框。

## Selector 规范

不引入复杂组件体系，只提供统一选择器能力。

- 固定少量选项：普通 Select。
- 动态列表：支持搜索的 Select。

适用：

- 清晰度、编码等固定配置使用普通选择。
- 用户、分组等动态数据使用搜索选择。

## 明确不做

- 不重构标签系统。
- 不增加首页大屏数据。
- 不重写下载流程。
- 不引入新的前端框架。

## 实施原则

先改基础组件，再调整页面，避免页面重复返工。
