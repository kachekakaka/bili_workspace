# 域名、HTTPS 与 QNAP 反向代理

## 推荐拓扑

```text
浏览器 / 手机
    │ HTTPS 443
    ▼
QNAP 反向代理（bili.example.com）
    │ HTTP 到 NAS 局域网地址和应用端口
    ▼
bili-workspace 容器
```

应用外部端口可在 `docker/.env` 中通过 `HTTP_PORT` 配置，例如 3398、3389 或 8080。公网只开放 443，不建议把应用端口同时直接暴露到互联网。

## QNAP 反向代理规则示例

```text
来源协议：HTTPS
来源主机：bili.example.com
来源端口：443
目标协议：HTTP
目标主机：NAS 局域网地址
目标端口：docker/.env 中的 HTTP_PORT
```

保留原始 `Host`，并转发标准 `X-Forwarded-For` 和 `X-Forwarded-Proto`。应用只信任 `TRUSTED_PROXY_IPS` 中明确列出的代理来源。

## Docker 配置

```env
BIND_IP=0.0.0.0
HTTP_PORT=3398
PUBLIC_BASE_URL=https://bili.example.com
TRUSTED_HOSTS=bili.example.com
TRUSTED_PROXY_IPS=实际代理来源IP
COOKIE_SECURE=true
ENABLE_HSTS=false
```

若同时使用局域网域名，可逗号分隔：

```env
TRUSTED_HOSTS=bili.example.com,bili.lan
```

禁止：

```env
TRUSTED_HOSTS=*
TRUSTED_PROXY_IPS=*
```

数字 IP 直接访问由 `BILI_ALLOW_IP_HOSTS` 控制；服务器模式默认允许数字 IP，但任意未列入白名单的域名仍会被拒绝。

## Windows 域名/局域网模式

编辑 `.env` 或运行 `configure_network.bat`：

```env
BILI_HOST=0.0.0.0
BILI_PORT=3398
```

域名和代理设置位于 `config/runtime.env`：

```env
BILI_PUBLIC_BASE_URL=https://bili.example.com
BILI_TRUSTED_HOSTS=bili.example.com
BILI_TRUSTED_PROXY_IPS=代理来源IP
BILI_COOKIE_SECURE=true
BILI_HSTS=false
```

非回环监听会自动强制网站管理员认证。

## HSTS

第一次部署保持关闭。确认以下项目全部正常后再设为 `true`：

- HTTPS 证书有效；
- 域名解析稳定；
- 登录和安全 Cookie 正常；
- 不再需要通过 HTTP 访问该域名。

错误启用 HSTS 会增加恢复难度。

## 额外建议

- 使用强管理员密码并定期更换；
- QNAP 防火墙仅允许必要来源；
- 能使用 VPN 时优先 VPN；
- 不在 URL、日志或截图中暴露初始化令牌和 Bilibili Cookie；
- 不把 `CONFIG_DIR` 映射到 Web 公共目录。
