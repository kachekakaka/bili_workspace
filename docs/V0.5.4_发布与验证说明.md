# bili_workspace V0.5.4 发布与验证说明

## 发布目的

V0.5.4 将项目恢复为标准、可直接克隆和审查的 Git 仓库，并固化 Windows、QNAP Docker、配置升级、持久化目录、局域网/域名访问和设备导出的边界。

## 关键变化

- 移除 bootstrap、Base64 源码分片和 Actions 自恢复方案；
- `main` 直接保存完整 Python、前端、测试、Docker 和 Windows 脚本；
- 所有可变配置采用 `.default` 模板，实际配置保持未跟踪；
- 缺失 JSON 字段递归补入，dotenv 缺失键增量追加，不覆盖已有值；
- 支持可配置 IP/主机名和 1–65535 端口，非回环监听强制认证；
- QNAP 固定映射 config/media/cache/tmp，数据库、分组和任务位于 config；
- Windows `setup.bat` 可从固定 Release 运行包补齐 BBDown、FFmpeg 和离线 wheelhouse，并校验 ZIP 与内部逐文件 SHA-256；
- 修复配置文件使用局域网主机名时可信 Host 未自动同步的问题，并加强主机名语法校验；
- 保留媒体库、分组、清晰度控制、浏览器播放、一次性设备导出和网页扫码登录；
- 增加长期产品需求与架构基线，避免需求仅保存在聊天上下文。

## Windows 完整包验证边界

完整 Windows 恢复包会额外进行：

- BBDown 和 FFmpeg 固定 SHA-256 校验；
- 发布清单逐文件校验；
- Cookie、旧虚拟环境、缓存、ffplay、ffprobe 和数据库扫描；
- ZIP CRC、路径穿越和符号链接检查。

当前构建环境不能原生执行 Windows EXE，因此 Windows 原生工具启动和真实 Bilibili 下载需要在 Windows 上运行 `verify.bat` 和一次有权保存的媒体下载完成最终验收。此前用户已确认相关 Windows 真实搜索、下载、混流、播放和目录操作可用，但 V0.5.4 仍应按新包再次自检。

## QNAP/Docker 验证边界

已完成 Compose、目录映射、环境变量、安全结构和 Shell 静态检查。实际 QNAP Container Station 构建、PUID/PGID、反向代理来源 IP、证书和 NAS 硬件性能需在目标 QNAP 上验收。
