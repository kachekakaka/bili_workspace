# 源码仓库、运行数据与 Windows 发布包

## Git 仓库保存什么

仓库用于保存可审查、可测试和可构建的源码：

```text
app/                    FastAPI 后端
web/                    响应式网页
config/*.default        配置模板
Dockerfile / compose.yaml
docker/                 QNAP/Docker 辅助脚本
BBDown_portable/        Windows 工具目录说明和许可证
tests/                  自动化测试
docs/                   部署、迁移、安全与验收文档
```

仓库不保存：

```text
.env、config/config.json、config/runtime.env、docker/.env
.venv
BBDown.data
SQLite 数据库
媒体、封面缓存、日志和临时导出
Windows BBDown.exe、ffmpeg.exe 和 wheelhouse
```

实际配置与运行数据均由 `.gitignore` 排除，更新源码不会覆盖它们。

## 默认模板升级机制

仓库跟踪：

```text
.env.default
config/config.json.default
config/runtime.env.default
docker/.env.default
```

程序和辅助脚本会创建对应实际文件。新版模板出现新字段时：

- JSON 对象递归补充缺失字段；
- dotenv 文件只追加缺失变量；
- 已有值、额外字段和用户选择保持不变；
- JSON 更新使用原子替换并创建 `.bak`；
- 符号链接目标会被拒绝；
- 损坏配置不会被模板静默覆盖。

从较早版本升级时，旧根目录 `config.json` 会在新目标不存在时复制到 `config/config.json`，随后再执行缺失字段合并。

## Windows 使用

### 现有完整包目录接入 Git

保留旧完整包中的：

```text
BBDown_portable/BBDown.exe
BBDown_portable/ffmpeg/bin/ffmpeg.exe
BBDown_portable/BBDown.data
```

这些文件不会被 Git 跟踪，更新源码时会保留。实际配置、数据库和下载目录同样不会被 `git pull` 覆盖。

### 全新源码克隆

1. 安装 64 位 Python 3.11、3.12 或 3.13；
2. 执行 `setup.bat`；
3. 执行 `verify.bat`；
4. 执行 `start.bat`。

源码仓库不带 Windows 下载器二进制。没有工具时，网站和媒体库仍可启动，但搜索、扫码登录、真实下载、混流和兼容转码不可用。将工具放到约定路径后重新运行 `verify.bat`。

后续可使用 `update.bat` 执行快进更新、依赖同步、配置增量升级和完整自检。

## 为什么不提交 `.venv`

`.venv` 包含创建机器的绝对路径、Python ABI、平台相关启动器和缓存。它不能可靠跨目录、跨 Python 小版本或从 Windows 复用到 Linux/Docker。即使放入 Git LFS，这些可移植性问题也不会消失。

项目使用锁定依赖文件和 `setup.bat` 在当前机器重建虚拟环境；Docker 使用镜像重建运行时。这比提交某台开发机生成的 `.venv` 更稳定。

## 大文件与 Git LFS

Git LFS 可以存储超过普通 Git 单文件限制的内容，但并不适合解决虚拟环境可移植性。第三方 BBDown/FFmpeg 可由固定发布版本重新获得，并在完整发布包或 Docker 构建中校验，因此源码历史不永久堆积这些二进制。

后续若项目出现自己维护、无法稳定重新生成的大型资产，再单独评估 Git LFS。

## Docker 使用

Docker 构建会安装系统 FFmpeg，并按目标架构下载固定的 BBDown Linux 版本。运行数据全部位于宿主机映射目录。

```bash
./docker/ensure-env.sh
vi docker/.env
./docker/verify-config.sh
./docker/build-and-start.sh
```

更新：

```bash
git pull --ff-only
./docker/build-and-start.sh
```

持久化目录不会随代码拉取或容器重建消失。

## 发布原则

正式 Windows 发布包应执行：

```text
工具哈希校验
敏感文件扫描
Python 编译
Ruff
pytest
前端 JavaScript 语法检查
Windows BBDown/FFmpeg 冒烟测试
ZIP 路径、CRC 和逐文件哈希验证
```

发布包不得包含真实 `.env`、`BBDown.data`、管理员初始化令牌、数据库、媒体文件、日志或用户配置。
