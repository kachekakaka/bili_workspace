# V0.7.0 功能与验收

## 1. 发布范围

V0.7.0 是前端内部结构整理版本。PR 1–8 按批准方案顺序实施，保持既有搜索、下载、任务、媒体库、账号、权限、Windows、Docker 和持久化行为。

明确不变：

- 数据库 schema 继续为 v4；
- 不修改 API 路径、字段和响应 envelope；
- 不修改下载算法、BBDown 调用语义和画质判定；
- 不修改管理员/普通用户权限矩阵；
- 不修改标签业务模型；
- 不修改 `config/`、`userdata/`、`downloads/` 和 Docker 三目录映射。

## 2. 最终前端架构

- 原生 `.mjs` ES Modules，静态 import，无 npm、bundle 或 transpile；
- `web/index.html` 只加载 `qrcode.min.js` 和唯一应用入口 `app/main.mjs`；
- 页面统一导出 `mount(root, context)` 并返回幂等 `dispose()`；
- Router 每次切页递增 generation，abort 上一页 controller，调用 dispose，只有当前 generation 可提交结果；
- API、Session、Context、Modal、Toast、Confirm 和 TaskStream 各一份；
- Dashboard/Tasks 共用一个 `/api/events`；
- CSS 为 tokens/base/components/pages 四层；
- 无 Legacy Bridge、`BiliEnhancements`、`BiliLegacyApp`、页面 MutationObserver、捕获阶段抢占或原生 prompt/confirm。

## 3. 行为兼容验收

### Search

- 首次只请求当前页；
- 当前页成功后最多预加载下一页一页；
- 精准/模糊只筛选当前页标题，不联网；
- 快速 Search → Library → Tasks 时旧响应不能覆盖当前路由；
- 在第 2 页创建任务后保留 Search 和当前页。

### Library

- 分页、关键词、分组、标签、无标签、编码、清晰度、观看状态和排序保持可用；
- 批量标签、下载、删除和跨页选择保持可用；
- 播放器支持 Range、分 P、拖动和续播；
- Groups/Tasks 跳转使用一次性路由状态，不依赖旧全局。

### Tasks 与账号

- 反复进入/退出 Tasks 只存在一个 SSE；
- 登出关闭 SSE 并清空共享状态；
- 用户/分组操作只使用应用内 Modal/Confirm；
- Account 双页签、设备会话、Bilibili 扫码和首次强制改密保持可用；
- 普通用户无权限 hash 自动回到 Download。

## 4. 自动化门禁

- Python 3.11、3.12、3.13；
- compileall、Ruff、完整 pytest；
- `node --check` 全部 `.js` / `.mjs`；
- `node --test tests/frontend/*.test.mjs`；
- Playwright：1920×1080、1440×900、1024×768、768×1024、390×844；
- 请求计数：标题筛选不联网、Library 单操作单请求、TaskStream 单连接；
- 最终架构：单入口、单 renderer、无 Legacy/Observer/prompt/confirm。

## 5. Windows 验收

### 干净 clone

1. 在全新目录克隆 `main`；
2. 运行 `verify.bat`；
3. 校验内置 Python、BBDown、FFmpeg；
4. 执行源码校验、compileall、Ruff、完整 pytest、`.js/.mjs` 语法和 Node tests；
5. 运行 `start.bat`，确认版本为 `0.7.0`。

### 原地升级

1. 停止 V0.6.2 服务；
2. 备份 `config/`、`userdata/`、`downloads/` 和敏感的 `BBDown.data`；
3. `git pull --ff-only origin main`；
4. 运行 `verify.bat`；
5. 运行 `start.bat`，核对用户、任务、分组、标签、作品库和 Bilibili 登录状态。

## 6. Docker / QNAP 验收

1. 保持 `CONFIG_DIR`、`USERDATA_DIR`、`MEDIA_DIR` 映射不变；
2. 拉取 `ghcr.io/kachekakaka/bili_workspace:v0.7.0` 或从 tag 本地构建；
3. 验证容器内 `from app import __version__` 为 `0.7.0`；
4. 验证旧数据库启动、三目录持久化和非 root 容器；
5. 核对健康检查、登录、搜索、下载、任务、媒体库和 Range 播放。

## 7. 数据库、迁移和回滚

V0.7.0 不增加数据库迁移，数据库 schema 继续为 v4。由 v0.6.2 升级时无需恢复或转换数据库，原 `userdata/` 可直接复用。

回滚到 `v0.6.2`：

1. 停止应用或容器；
2. 备份当前 `config/`、`userdata/` 和媒体目录；
3. 切换到 tag `v0.6.2`；
4. 保持原持久化目录不变并重新启动；
5. 运行对应版本自检，核对用户、会话、任务、作品、标签、删除历史和播放记录。

由于 V0.7.0 不改变 schema 或 API，正常情况下不需要数据库降级；但禁止在未备份的情况下直接覆盖故障现场。
