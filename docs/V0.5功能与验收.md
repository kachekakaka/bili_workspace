# V0.5.x 功能与验收清单

本清单对应当前 `main`。带具体版本号的历史发布报告只说明当时状态，验收新代码应以本文件为准。

## 核心功能

- Windows 本地运行与 QNAP Docker 服务器运行；
- 原始、精准和模糊搜索；精准/模糊均只调用一次 Bilibili 原始搜索，再只按标题过滤；
- 搜索页默认显示 10 个页码，可继续增加 10 页；
- 搜索状态包含未下载、排队中、下载中、已下载、失败、取消和已删除；
- 默认屏蔽已下载和已删除；
- NAS 媒体库、标签、分组、无标签筛选、排序和批量操作；
- 删除作品后保留不可见的独立删除历史；
- 清晰度预检、实际码流复核、真实下载进度和日志；
- 任务原 ID 重试、编辑画质、批量暂停/继续/取消/重试/删除；
- 浏览器 Range 播放、观看记录、多文件和多分 P 切换；
- 手动生成 H.264/AAC 兼容副本；
- 一次性设备导出：临时下载、浏览器传输、成功清理、断线保留和 TTL 清理；
- 网站管理员认证、CSRF、安全 Cookie、登录节流、密码修改和会话撤销；
- 网页 Bilibili 二维码登录；
- SQLite 持久化作品、标签、删除历史、任务、分组、观看记录和导出状态；
- SSE 实时任务更新和封面缓存。

## Windows 验收

1. 全新克隆或解压后运行 `start.bat`，确认便携 Python、BBDown 和 FFmpeg 自动就绪。
2. 运行 `verify.bat`，确认源码、Python、依赖、BBDown 和 FFmpeg 冒烟测试通过。
3. 搜索公开作品，分别验证原始、精准和模糊模式。
4. 输入 `(测试 原因)` 一类多词条件，确认精准只保留标题含全部词，模糊保留标题含任意词。
5. 确认搜索默认显示前 10 个页码，并可点击“再显示10页”。
6. 选择分组或新建分组，预览画质并完成下载。
7. 确认创建任务后仍停留在搜索或创建下载页面。
8. 任务中心显示标题、BV、实际清晰度、编码、阶段、大小、速度、ETA、时长和分 P。
9. 制造一个失败任务，修改清晰度后重试，确认任务 ID 不变且列表不新增重复记录。
10. 验证批量暂停、继续、取消、重试、删除和清理失败/取消任务。
11. 作品进入作品库并能播放、拖动、保存观看位置和切换多 P 文件。
12. 通过分组 chip、标签 chip、“无标签”和排序字段筛选作品。
13. 修改作品分组，确认不重新下载文件。
14. 删除作品，确认作品库不再显示；关闭搜索屏蔽后，搜索结果标记“已删除”。
15. 重新下载已删除作品，确认成功后显示“已下载”。
16. 从作品库下载到当前设备，媒体库原文件仍存在。
17. 创建“设备导出”任务，浏览器完整下载后临时文件状态变为已清理。
18. 中断大文件设备导出，任务仍允许再次下载，并最终由 TTL 清理。
19. 将监听地址设为 `0.0.0.0`，确认设置页显示局域网访问地址，手机可访问并要求管理员登录。

## QNAP / Docker 验收

1. `docker/verify-config.sh` 通过。
2. `docker compose --env-file docker/.env config` 通过。
3. `./docker/build-and-start.sh` 成功，容器健康状态为 healthy。
4. 三个宿主机映射目录正确：`CONFIG_DIR`、`USERDATA_DIR`、`MEDIA_DIR`。
5. 容器内部对应 `/data/config`、`/data/userdata`、`/downloads`。
6. 首次初始化令牌只能使用一次。
7. 未登录访问业务 API 返回 401；修改 API 无 CSRF 返回 403。
8. 域名 HTTPS 登录后 Cookie 带 Secure、HttpOnly、SameSite。
9. 网页 Bilibili 扫码成功，浏览器接口与日志不出现完整 Cookie。
10. 手机可搜索、播放、暂停、拖动、继续观看和下载文件。
11. NAS 重启后媒体、分组、标签、删除历史、播放记录与任务历史存在；运行中任务标记为中断。
12. 删除作品后媒体库不显示，搜索可识别已删除状态。
13. 设备导出完整发送后清理，中断后保留并可重试。
14. 大文件播放期间应用内存不随文件大小线性增长。
15. 更新镜像或重建容器后，三个映射目录中的数据仍存在。

## 数据目录验收

Windows：

```text
config/      配置和标签定义
userdata/    SQLite、删除历史、任务、索引、日志、缓存和临时状态
downloads/   媒体文件
```

Docker：

```text
/data/config
/data/userdata
/downloads
```

确认数据库至少包含作品库、标签关系、任务快照和 `deleted_media` 删除历史；删除记录不得作为普通作品出现在作品库中。

## 安全验收

- 不以 root 运行，不使用 privileged，不挂载 Docker socket；
- 根文件系统只读，Linux capabilities 全部删除；
- Host 白名单与可信代理不使用通配符；
- 媒体、导出、删除和打开目录均通过服务器保存的 ID/相对路径定位；
- 路径穿越、符号链接、UNC、盘符和目录越界测试通过；
- 发布包不含 `BBDown.data`、数据库、初始化令牌、实际 `.env`、运行日志或真实 Cookie。
