# V0.5.x 功能与验收清单

## 核心功能

- Windows 本地运行与 QNAP Docker 服务器运行。
- NAS 媒体库、分组管理、搜索、清晰度预检、真实下载进度和日志。
- 浏览器 Range 播放、观看记录、多文件切换和原文件下载。
- 手动生成 H.264/AAC 兼容副本。
- 一次性设备导出：临时下载、浏览器传输、成功清理、断线保留和 TTL 清理。
- 网站管理员认证、CSRF、安全 Cookie、登录节流、密码修改和会话撤销。
- 网页 Bilibili 二维码登录。
- SQLite 数据持久化、V0.4 索引导入、SSE 增量更新和封面缓存。

## Windows 验收

1. 全新解压后运行 `verify.bat`。
2. BBDown 与 FFmpeg 冒烟测试通过。
3. 搜索公开作品并预览清晰度。
4. 选择分组或新建分组，完成下载。
5. 任务显示标题、BV、实际清晰度、分辨率、编码和进度。
6. 作品进入作品库并能播放、拖动和保存观看位置。
7. 从作品库下载到当前设备，NAS 原文件仍存在。
8. 创建“设备导出”任务，浏览器下载完成后临时文件状态变为已清理。
9. 中断大文件设备导出，任务仍允许再次下载，并最终由 TTL 清理。

## QNAP 验收

1. `docker/verify-config.sh` 通过。
2. `docker compose build --pull` 和 `docker compose up -d` 成功。
3. `/healthz` 正常，容器健康状态为 healthy。
4. 首次初始化令牌只能使用一次。
5. 未登录访问业务 API 返回 401；修改 API 无 CSRF 返回 403。
6. 域名 HTTPS 登录后 Cookie 带 Secure、HttpOnly、SameSite。
7. 网页 Bilibili 扫码成功，浏览器接口与日志不出现完整 Cookie。
8. 手机可播放、暂停、拖动、继续观看和下载文件。
9. NAS 重启后媒体、分组、播放记录与任务历史存在；运行中任务标记为中断。
10. 设备导出完整发送后清理，中断后保留并可重试。
11. 大文件播放期间应用内存不随文件大小线性增长。
12. `/data/config`、`/data/media`、`/data/cache`、`/data/tmp` 均落在宿主机映射目录。

## 安全验收

- 不以 root 运行，不使用 privileged，不挂载 Docker socket。
- 根文件系统只读，Linux capabilities 全部删除。
- Host 白名单与可信代理不使用通配符。
- 媒体、导出、删除和打开目录均通过服务器保存的 ID/相对路径定位。
- 路径穿越、符号链接、UNC、盘符和目录越界测试通过。
- 发布包不含 `BBDown.data`、数据库、初始化令牌、`.env`、运行日志或真实 Cookie。
