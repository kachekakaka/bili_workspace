## 当前关键约定

### 目录

Windows / 源码目录：

```text
config/      配置模板生成的实际配置、标签定义
userdata/    SQLite、删除记录、任务快照、下载索引、日志、缓存和临时状态
downloads/   只保存媒体文件
```

Docker 容器：

```text
/data/config     配置和 Bilibili 凭据
/data/userdata   SQLite、索引、任务、删除记录、日志、缓存和临时状态
/downloads       媒体文件
```

### 根目录和脚本

根目录只保留 Windows 常用入口 `start.bat`、`update.bat`、`verify.bat`。内部运行时、备用命令行和开发校验脚本统一放在 [`scripts/`](../scripts/README.md)。

### 当前搜索行为

- 搜索区只请求 Bilibili 原始当前页结果，每页 20 条；
- 精准/模糊是浏览器当前页标题二级筛选，不会因为切换模式或修改筛选词而联网；
- 首次只加载当前页，成功渲染后空闲时最多预加载下一页 1 页，不会后台抓取十页；
- WBI 密钥缓存 10 分钟，原始搜索页缓存 3 分钟，“刷新B站结果”只刷新当前页；
- 标签、下载状态和删除状态随搜索响应一起返回；
- “屏蔽已下载和已删除”默认隐藏两类作品。

### 当前 UI 行为

- 桌面端普通按钮、输入框和下拉框统一为 40px，移动端触控控件不低于 44px；
- 用户显示名、临时密码和分组重命名使用站内 Modal，不再调用浏览器 `prompt`；
- 账号页分为“Bilibili 登录”和“网站账号与设备”，网站会话最多 10 个有效 Token；
- 用户、分组等动态选项超过 8 个时使用带搜索的选择列表；
- 设置页默认显示常用项，高级下载策略折叠展示。

### 删除记录

删除作品会移除作品库记录和媒体文件，但在 `userdata/bili_workspace.db` 的独立删除记录表中保留历史。删除历史不作为普通标签，也不会重新出现在作品库；关闭搜索屏蔽后会显示“已删除”。主动重新下载成功后，该删除记录会清除。

### 标签和分组

- 标签定义位于 `config/tags.json`，默认模板为 `config/tags.json.default`；
- 标签关系、分组、作品库和删除历史保存在 SQLite；
- 作品库支持分组和标签 chip 筛选，包括“无标签”；
- 作品可直接修改分组，不需要重新下载。
