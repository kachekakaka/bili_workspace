# 源文件与恢复清单

本文件记录项目可恢复来源和长期保留资产，避免关键版本只存在于聊天记录、临时目录或失效链接中。

## 1. 当前可信源码

当前可运行源码以 GitHub 仓库 `main` 为准：

```text
https://github.com/kachekakaka/bili_workspace
```

`main` 必须直接包含可审查的 Python、前端、测试、Docker、Windows 脚本和文档，禁止使用 Base64 分片或运行时恢复源码。

重要版本应额外保留：

```text
源码 ZIP
完整 Git Bundle
对应 Git 标签或提交 SHA
发布验证报告
SHA256SUMS.txt
```

## 3. 当前 Windows 集成运行资产

Windows x64 运行环境直接以普通 Git 对象保存在：

```text
vendor/windows/python-runtime.pack
vendor/windows/media-runtime.pack
vendor/windows/runtime-manifest.json
```

当前清单记录：

```text
Python 3.13.14
python-runtime.pack SHA-256 0b20c48727a144520f8fd1676a2252079fe14c4483df669c5490ba54d36f1313
media-runtime.pack  SHA-256 1b8ac6956c7e08110b477c8a132709a10c0d0c75360c14954397c11b3c836632
```

运行资产包含：

```text
Portable Python 和锁定依赖
BBDown 1.6.3
FFmpeg
第三方许可证
包内逐文件清单
```

`start.bat` / `bootstrap.bat` 会校验 pack、路径和内部清单，再解压到被 Git 忽略的 `.runtime/` 与 `BBDown_portable/`。恢复时不需要复制旧 `.venv` 或旧 `.runtime`。

## 4. 当前运行数据

Windows：

```text
config/      配置和标签定义
userdata/    SQLite、任务、删除历史、索引、日志、缓存和临时状态
downloads/   媒体文件
BBDown_portable/BBDown.data（敏感）
.env（监听地址和端口）
```

QNAP / Docker 宿主机：

```text
CONFIG_DIR
USERDATA_DIR
MEDIA_DIR
```

容器内对应：

```text
/data/config
/data/userdata
/downloads
```

完整恢复必须保留数据库和媒体的一致性。在线复制 SQLite 时同时保留 `.db`、`-wal` 和 `-shm`。

## 5. 数据库中需要保留的内容

`userdata/bili_workspace.db` 包含：

- 管理员、会话和审计记录；
- 作品库和媒体文件信息；
- 分组和观看进度；
- 标签定义同步结果与作品标签关系；
- 任务快照、导出和转码状态；
- `deleted_media` 删除历史。

删除历史不作为作品显示，但搜索依赖它标记“已删除”。只备份 `downloads/` 而不备份数据库，会丢失标签、删除历史、任务和播放记录。

## 6. 每次正式发布必须保留

```text
1. 可直接审查的 GitHub main 源码树
2. 对应版本标签或提交 SHA
3. 源码 ZIP
4. 完整 Git Bundle
5. Windows 集成运行 pack 和清单
6. Windows 完整恢复包
7. QNAP Docker 部署说明或包
8. SHA256SUMS.txt
9. 发布验证报告
10. 产品需求与架构基线
11. 当前文档索引
```

Git Bundle、Windows 完整恢复包、运行数据备份和 SHA256SUMS 应额外保存到用户自己的 NAS 或云盘；GitHub 不是唯一备份。

## 7. 恢复优先级

1. 从 GitHub `main` 或已验证标签恢复源码；
2. 校验 `vendor/windows/runtime-manifest.json` 并重新生成本地运行目录；
3. 恢复同一时间点的 `config`、`userdata` 和 `downloads`；
4. 恢复敏感的 `BBDown.data`；
5. 运行 `start.bat` 或 Docker 启动脚本；
6. 运行验证并核对作品、标签、删除历史、分组、任务和播放记录。

任何备份或发布包都必须排除真实登录 Cookie、管理员初始化令牌、未加密密码和用户媒体，除非它本身就是受保护的私人运行数据备份。
