# 源文件与恢复清单

本文件记录项目可恢复来源和长期保留资产，避免关键版本只存在于聊天记录、临时目录或失效链接中。

## 用户提供的基线文件

以下三个文件是 V0.5.x 仓库重建的原始依据：

```text
bili_workspace_v0.5.0_GitHub源码(1).zip
SHA-256 0068d0547c178f759e2f9ffb0143b95e9b1c7f6c61f693167e10b59dd123e33d

bili_workspace_v0.5.0_QNAP_Docker部署包(1).zip
SHA-256 59f3f9fe1b73afd9e9bee308f92fe3f535767490f96f4163889b9c86ee1d891e

bili_workspace_v0.5.0_Windows_NAS媒体库完整版(1).zip
SHA-256 1222ed826fe932aa99705f0e425be1ea9fbf0b5fcb7c2f26ab58d26b3584f72a
```

这些原始包不得重新发布为最新源码，但应在用户自己的云盘或 NAS 备份中只读保留，用于核对第三方二进制、离线 wheelhouse 和早期迁移数据。

## V0.5.4 Windows 运行资产

Git 普通提交不保存第三方 EXE、wheelhouse 或 `.venv`。Windows 开箱初始化使用 GitHub Release 资产：

```text
bili_workspace_v0.5.4_windows_runtime.zip
SHA-256 e084d898d3c7405488380bb4bda8cd11786158bf2af5e974cf455abf1615f8c4
```

运行资产包含：

```text
BBDown_portable/BBDown.exe
BBDown_portable/ffmpeg/bin/ffmpeg.exe
wheelhouse/*.whl
第三方许可证
runtime_manifest.sha256
```

`setup.bat` 和 `tools/bootstrap_windows_runtime.py` 会校验整个 ZIP 与内部逐文件哈希。Release 不可访问时，可把该 ZIP 放在仓库根目录离线安装。

## 每次正式发布必须保留

```text
1. 可直接审查的 GitHub main 源码树
2. 对应版本标签
3. 源码 ZIP
4. 完整 Git Bundle
5. Windows 运行资产
6. Windows 完整恢复包
7. QNAP Docker 部署包
8. SHA256SUMS.txt
9. 发布验证报告
10. 产品需求与架构基线
```

其中 Git Bundle、Windows 完整恢复包和 SHA256SUMS 应额外保存到用户自己的 NAS 或云盘；GitHub 不是唯一备份。

## 运行数据的备份边界

Windows 应备份：

```text
config/
downloads/
BBDown_portable/BBDown.data（敏感，需加密）
```

QNAP/Docker 应备份：

```text
CONFIG_DIR
MEDIA_DIR
```

`CACHE_DIR` 可重建，`TEMP_DIR` 不应作为长期备份。任何备份或发布包都必须排除真实登录 Cookie、管理员初始化令牌和未加密密码。
