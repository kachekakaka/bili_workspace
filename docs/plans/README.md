# 已批准开发计划

本目录保存已经确认、但尚未全部进入当前 `main` 的开发基线。

## 当前计划

- [V0.6.0 多用户、会话 Token、搜索与界面开发基线](V0.6.0_多用户搜索与会话方案.md)

## 使用规则

- `docs/需求落实清单.md` 只描述当前 `main` 已实现的功能；
- 本目录描述后续开发目标，不得在功能尚未合并时当成现有能力；
- 开发者必须从最新 `main` 建立 `agent/*` 分支；
- 一次只完成计划中的一个 PR 阶段；
- 功能、测试和当前主线文档必须在同一个 PR 中同步；
- CI 全部通过后再合并到 `main`；
- 旧的版本性验收、发布报告和已被替代的方案应移动到 `docs/archive/`。

当 V0.6.0 全部完成后，应把计划状态改为“已实现”，并新增正式的 `docs/V0.6功能与验收.md`。
