# 已批准开发计划

本目录保存已经确认的开发计划历史和仍未进入当前 `main` 的开发基线。

## 已完成计划

- V0.6.0 多用户、会话 Token、搜索与界面开发基线：已完成。
  - PR 1–5 已全部合并。
  - 正式验收文档已进入 `docs/V0.6功能与验收.md`。

- V0.6.2 UI/UX 修正版：已完成。
  - UI/UX 规范、问题清单和自动化回归已进入当前主线。
  - 后续 UI 优化从新的版本计划开始，不再修改本计划。

## 使用规则

- `docs/需求落实清单.md` 只描述当前 `main` 已实现的功能。
- 本目录中的未完成计划才可以作为后续开发依据。
- 新开发必须从最新 `main` 建立 `agent/*` 或 `feature/*` 分支。
- 一次只完成计划中的一个 PR 阶段。
- 功能、测试和当前主线文档必须在同一个 PR 中同步。
- CI 全部通过后再合并到 `main`。
- 旧版本性验收、发布报告和被替代方案应移动到 `docs/archive/`。
