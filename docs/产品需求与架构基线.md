# bili_workspace 产品需求与架构基线

本文件是后续开发、审查和发布的长期依据。新增需求落地时，应同时更新本文件、测试和 `CHANGELOG.md`，避免只存在于聊天记录中。

## 1. 产品定位

`bili_workspace` 同时提供：

- Windows 本机下载与媒体库；
- QNAP NAS / Docker 长期运行服务；
- 手机、平板和桌面浏览器访问；
- 私人媒体库、分组、在线播放、设备导出和下载任务管理。

仅用于用户有权下载、保存和播放的内容。

## 2. 下载与清晰度

- 搜索、预览、任务、作品库必须同时显示标题和 BV 号；
- 下载前预览清晰度、分辨率、编码、帧率、分 P 和估算大小；
- 支持最低清晰度与明确目标档位；
- 下载前预检和下载后实际码流复核必须同时通过；
- 不允许任务显示成功却静默保存低于门槛或与指定档位不一致的文件；
- 下载日志保留原始诊断信息，同时提供结构化阶段、百分比、速度和 ETA；
- 下载核心应保持可替换，避免与已停止维护的单一第三方工具深度耦合。

## 3. 分组

- 下载时可搜索并选择已有分组；
- 可在下载表单内立即创建并选中新分组；
- 提供独立分组页，可查看作品数量、空间、任务和最近更新；
- 支持重命名、合并、移动作品、批量移动和删除空分组；
- 显示名称与物理目录标识解耦，普通重命名不搬动大型文件；
- 删除分组不得隐式删除媒体文件。

## 4. 媒体库与播放

- 手机、平板和桌面浏览器均可使用；
- 支持 `HEAD`、字节 `Range`、`206`、`416`、`ETag`、拖动和断点续播；
- 服务端不得把整个大文件一次性读入内存；
- 默认优先直接播放原始文件；
- 浏览器不兼容时由用户手动生成 H.264/AAC MP4 兼容副本；
- 兼容副本不得覆盖原始高质量文件；
- 支持多分 P、字幕、播放位置和最近观看。

## 5. 导出到当前设备

下载目标包括：

1. 永久保存到 NAS / 本机媒体库；
2. 临时下载后导出到当前浏览器设备。

设备导出规则：

- NAS 临时下载、混流和必要打包完成后再提供下载；
- 使用流式响应，不把大文件整体载入内存；
- 服务器完整发送全部响应字节后立即删除专属临时文件；
- 浏览器中断、断网或未读完时不立即删除，允许重试；
- 中断文件由可配置 TTL 清理兜底；
- 只能通过数据库对象 ID 访问，浏览器不能提供任意服务器路径；
- 清理逻辑只能作用于设备导出专属临时目录，不得触碰媒体库。

## 6. 认证与 Bilibili 扫码

- 回环本机模式可选择不启用网站管理员认证；
- 监听非回环地址时必须自动切换服务器模式并强制认证；
- 基础版本采用单管理员，无开放注册和匿名公网媒体；
- 需要服务端会话、密码哈希、CSRF、登录限速、安全 Cookie、可信 Host/代理和审计日志；
- 网页可创建并显示 Bilibili 登录二维码；
- 完整 Bilibili Cookie 只保存在服务器秘密目录，不返回浏览器、不进入日志和发布包；
- 提供二维码过期、刷新、扫码确认、退出登录和凭据清理。

## 7. 网络与域名

- 监听地址不能写死；支持 `127.0.0.1`、`0.0.0.0`、指定局域网 IP、IPv6 和合法主机名；
- 端口支持 `1–65535`，包括 3389，但应提示与 Windows 远程桌面冲突的可能；
- 手机局域网访问使用电脑/NAS 局域网 IP 加端口；
- 公网推荐使用独立 HTTPS 子域名和 QNAP 反向代理；
- 公网只开放 443，不建议直接裸露应用端口；
- 转发头只信任明确的反向代理来源 IP。

## 8. 配置模板与升级

Git 跟踪：

```text
.env.default
config/config.json.default
config/runtime.env.default
docker/.env.default
```

Git 不跟踪对应实际文件。启动和更新必须：

- 实际配置不存在时从同名 `.default` 创建；
- JSON 配置递归补入模板新增而实际缺少的字段；
- dotenv 配置只追加缺失变量；
- 保留已有值、额外字段、空字符串、`0`、`false` 和用户选择；
- 使用原子写入并保留备份；
- 拒绝符号链接配置目标；
- 损坏配置不得被模板静默覆盖；
- 需要迁移时提供显式版本与可回滚迁移。

## 9. Docker/QNAP 持久化边界

容器固定使用：

```text
/data/config  配置、SQLite、任务、分组、管理员、会话、观看进度和 Bilibili 凭据
/data/media   永久媒体文件与下载索引
/data/cache   封面与兼容播放副本
/data/tmp     下载、混流、转码和设备导出临时文件
```

宿主机目录通过 `docker/.env` 映射。更新 Git、重建镜像或重建容器不得清空这些目录。

容器应：

- 非 root 运行；
- 不使用 privileged；
- 不挂载 Docker socket；
- 根文件系统只读；
- 丢弃不需要的 capabilities；
- 有健康检查、日志轮转和优雅停止。

## 10. 性能与可靠性

- 任务变化优先通过 SSE 增量推送，保留轮询后备；
- 作品库使用数据库服务端筛选、排序和分页；
- 封面懒加载并受控缓存；
- 下载、转码和缩略图并发必须有上限；
- 开始任务前检查临时盘和媒体盘剩余空间；
- 任务、分组、作品库和观看记录重启后保留；
- 运行中任务在异常重启后标记中断或恢复排队，不能误判成功；
- 日志、封面缓存、兼容副本和导出临时文件必须有容量或 TTL 边界。

## 11. Git 与发布

- GitHub `main` 必须直接包含可审查源码树，禁止使用 Base64 分片或 Actions 自解包恢复源码；
- 正常 CI 只执行源码结构、编译、静态检查、测试和前端语法检查；
- 实际配置、凭据、数据库、媒体、日志和缓存不得提交；
- `.venv` 不提交，因为包含绝对路径、Python ABI 和平台相关入口；
- Windows 完整包可包含经过哈希验证的 BBDown、FFmpeg 和离线 wheelhouse；
- 源码仓库通过锁定依赖、GitHub Release 运行资产和安装脚本重建环境；
- Release 运行资产必须固定版本和 SHA-256，内部还需逐文件清单；网络不可用时支持将同名资产放在仓库根目录离线安装；
- 每次重要发布保留源码 ZIP、完整 Git Bundle、Windows 恢复包、校验文件和验证报告。

## 12. 当前明确不纳入 V0.5.x

- 开放注册；
- 匿名公网分享；
- 多管理员角色；
- 自动为所有媒体生成多码率 HLS；
- 强依赖 NAS GPU；
- 自动删除原始媒体；
- Plex/Jellyfin 深度集成；
- 多节点任务调度。
