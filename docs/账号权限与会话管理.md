# 账号权限与会话管理

本文件描述当前 `main` 已经实现的 V0.6 账号、会话和多用户前端。

## 账号类型

系统只允许一个启用的管理员，普通用户可由管理员创建。登录账号为 3–32 位 ASCII，必须以英文字母开头，后续只允许字母、数字、点、下划线和短横线；比较时不区分大小写。中文显示名为 2–12 个汉字，可重复。

正式密码为 10–64 位可见 ASCII，至少包含一个英文字母和一个数字，不允许空格、中文或控制字符。密码继续使用 scrypt 哈希保存。

## 首次初始化

Windows 回环地址上的全新数据库自动创建：

```text
账号：admin
临时密码：123456
显示名：管理员
```

该临时密码只能从 `127.0.0.1` 或 `::1` 登录。登录后只能查看认证状态、修改密码或退出；修改前拒绝其他 API，也拒绝把应用切换为非回环监听。

Docker、NAS 和非回环部署继续使用 `BILI_BOOTSTRAP_TOKEN` 或 `config/bootstrap-token.txt` 中的一次性令牌。初始化成功后令牌立即失效，不开放远程 `admin / 123456`。

## 独立会话 Token

每次登录生成一个新的高熵 Token，只通过 HttpOnly Cookie 写入浏览器。HTTP 使用 `bili_session`，HTTPS 使用 `__Host-bili_session` 并启用 Secure；均为 `SameSite=Lax; Path=/`。数据库只保存 SHA-256 哈希，明文 Token 不进入 API JSON、SQLite 或审计日志。

每用户最多 10 个有效 Token。有效会话要求未撤销、未过期且用户未禁用。创建第 11 个会话时，在同一 SQLite 事务中保留新会话，并按 `last_seen_at ASC, created_at ASC` 撤销最久未连接的会话。成功认证请求最多每 60 秒更新一次 `last_seen_at`。

用户可列出自己的有效设备、撤销其他单个设备、退出其他设备或退出当前设备。退出当前设备不会影响同一用户的其他会话。

## CSRF 与改密

POST、PUT、PATCH 和 DELETE 请求必须携带当前会话自己的 `X-CSRF-Token`。其他会话的 CSRF Token 无效。

修改密码会：

1. 验证当前密码和新密码规则；
2. 轮换当前会话 Token 与 CSRF Token；
3. 撤销该用户其他全部会话；
4. 清除 `must_change_password`。

管理员重置普通用户临时密码、禁用用户或主动撤销会话时，会立即撤销该用户所有有效会话。

## 管理员用户 API

当前后端提供：创建普通用户、修改中文显示名、禁用/启用、设置临时密码、查看有效会话数量和撤销全部会话。不能创建第二个管理员，也不能禁用当前管理员。

普通用户可使用认证、本人会话、下载、任务、本人日志、本人 SSE 和本人设备导出接口；搜索、作品库、分组、Bilibili 账号、系统设置和管理员 API 均由后端返回 403。网页侧边栏和手机底栏也只显示“下载”和“任务”，但前端隐藏不替代后端权限检查。

## 用户菜单和管理员用户管理

所有已登录用户的顶部菜单显示中文显示名、登录账号和角色，并提供：修改中文显示名、修改密码、管理登录设备和退出登录。修改密码成功后前端接收轮换后的 CSRF，其他设备立即失效。

管理员侧边栏提供“用户管理”页面。桌面使用表格，手机使用卡片，可创建普通用户、修改显示名、禁用/启用、设置临时密码、撤销全部会话和查看该用户任务。普通用户不会加载用户管理、分组、标签或 Bilibili 账号接口。

## 数据库迁移和回滚

schema 升级前使用 SQLite `backup()` 在以下目录生成一致性备份：

```text
userdata/backups/bili_workspace-v<旧版本>-YYYYMMDD-HHMMSS.db
```

只保留最近 3 份。迁移在 `BEGIN IMMEDIATE` 事务中完成，失败立即回滚且应用不开放 HTTP 服务；成功后执行 `PRAGMA foreign_key_check`。旧管理员密码保持不变，旧会话统一撤销一次。

回滚时停止应用，恢复对应备份文件并切换回旧版本代码后再启动。不得在应用运行时直接覆盖 SQLite 主文件。
