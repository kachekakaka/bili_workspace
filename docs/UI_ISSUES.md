# UI Issues

## V0.6.2 处理状态

本清单记录截图 Review 中发现的问题。V0.6.2 范围内的 P0、P1 和 P2 项已实现并纳入自动化回归。

## P0（已完成）

### 控件尺寸不统一

原现象：

- 下拉框高度明显大于按钮；
- 输入框和操作按钮视觉比例不一致。

处理结果：

- 桌面端普通 Button/Input/Select 统一为 40px；
- 紧凑按钮使用 32px；
- 移动端触控控件不低于 44px；
- 五档固定视口继续检查横向溢出和控件重叠。

### Prompt 交互

原现象：

- 用户修改显示名、设置临时密码和分组重命名使用浏览器 prompt。

处理结果：

- 三类操作统一进入 `modalRoot`；
- 捕获阶段拦截历史事件，保证原生 `window.prompt` 不再被调用；
- Playwright 注入 prompt 探针验证没有回退到浏览器弹窗。

### 账号页面职责混合

原现象：

- Bilibili 登录和网站用户密码管理同时显示，概念混淆。

处理结果：

- 账号页拆成“Bilibili 登录”和“网站账号与设备”页签；
- 网站页展示登录账号、中文显示名、修改密码和有效 Token/设备；
- Bilibili 页只负责扫码、Cookie 和登录状态。

## P1（已完成）

### 下拉选择

原现象：

- 动态数据使用普通下拉不易查找。

处理结果：

- 清晰度、编码等固定少量选项继续使用原生 Select；
- 用户、分组、任务拥有者等动态选项超过 8 项时，切换为带搜索框的两列选择列表；
- 不引入第三套复杂选择器或新前端框架。

### 设置页面

原现象：

- 高级配置和普通配置混合。

处理结果：

- 地址、端口、媒体目录、默认分组和默认清晰度保持默认展开；
- 超时、轮询、画质和编码策略放入“高级设置”折叠区；
- 折叠内容显式从布局中隐藏，避免与保存按钮重叠。

## P2（已完成）

- 页面顶部和卡片内边距适度压缩；
- 下载 textarea 调整为约 120px；
- 筛选、账号和会话区域使用统一控件尺寸；
- 桌面、平板和手机固定视口无横向滚动或可见控件重叠。

## 非目标

- 标签系统重构；
- 下载流程重写；
- 数据库、API 或权限模型修改；
- 大规模前端架构调整。
