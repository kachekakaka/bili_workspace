# bili_workspace V0.5.0 发布说明与验证报告

> 历史发布记录。发布日期：2026-07-17。当前 `main` 已显著扩展搜索、标签、任务和删除历史，并将目录整理为 `config`、`userdata`、`downloads`。当前使用和部署请从 `docs/README.md` 开始。

版本定位：Windows 本地下载器 + QNAP/NAS 私人媒体库 + 浏览器在线播放与一次性设备导出。

## 1. V0.5.0 当时范围

V0.5.0 在 V0.4.0 的分组、标题/BV 双标识、清晰度预览和低清晰度硬拦截基础上，完成：

- Windows 本地模式与 NAS/Docker 服务器模式共用后端、前端和数据模型；
- SQLite WAL 持久化管理员、会话、逻辑分组、媒体库、播放进度、任务快照、设备导出和审计日志；
- 响应式作品库、分组管理、任务中心、账号页和设置页；
- 浏览器标准 Range/HEAD/206/416 媒体流；
- 原文件优先播放，不兼容时手动生成 H.264/AAC/MP4 兼容副本；
- 下载目标可选“保存到媒体库”或“导出到当前设备”；
- 设备导出完整发送后立即清理，中断保留到 TTL 到期或用户手动放弃；
- 下载时选择或新建分组，分组页可浏览、重命名、合并和删除空分组；
- NAS 模式强制管理员认证、CSRF、可信 Host/代理、安全 Cookie、登录节流、审计和会话撤销；
- 网页 Bilibili 扫码登录，完整 Cookie 只写入服务器 `BBDown.data`；
- Docker 当时采用四目录映射和安全容器设置。

## 2. 设备导出的准确语义

```text
NAS 临时目录下载和混流
→ 浏览器以流式响应接收
→ 应用完成全部响应字节发送
→ 删除导出文件、源产物和临时索引
```

- 不把大文件整体读入内存；
- HEAD 请求不会触发清理；
- 浏览器中断或响应迭代未完成时不会触发完成清理；
- 中断文件默认保留到 TTL，可重新下载或手动放弃；
- HTTP 只能证明服务器完成响应发送，不能证明浏览器最终成功写入设备磁盘。

## 5. 当前主线与 V0.5.0 的差异

当前 `main` 额外包含：

- 原始、精准和模糊搜索，精准/模糊只对一次原始搜索返回的标题进行过滤；
- 10 页起步的分页和继续增加 10 页；
- 标签、无标签、分组 chip 和作品直接改分组；
- 作品库多字段升降序排序；
- 任务原 ID 重试、画质编辑和批量任务控制；
- 独立 `deleted_media` 删除历史；
- 默认屏蔽已下载和已删除；
- Windows / Docker 三类持久化目录。

当前目录：

```text
Windows：config/、userdata/、downloads/
Docker：/data/config、/data/userdata、/downloads
```

旧版“四个目录”只用于理解 V0.5.0 发布时状态，不应继续用于新部署。
