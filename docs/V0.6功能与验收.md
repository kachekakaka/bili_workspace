# bili_workspace V0.6.0 功能与验收

> 发布日期：2026-07-20  
> 应用版本：`0.6.0`  
> 数据库 schema：`v4`

V0.6.0 完成五个阶段：Bilibili 原始当前页搜索与标题二级筛选、独立网站账号和 HttpOnly 会话 Token、任务/日志/SSE/导出所有权、普通用户双页面与管理员用户管理，以及 Windows/Docker/数据库最终发布验收。

## 2. 搜索验收

- 点击搜索只请求 Bilibili 当前页，每页 20 条；
- 精准/模糊只对当前页标题做二级筛选，切换模式或修改筛选词产生 0 次网络请求；
- 首次搜索最多 WBI + 搜索 2 次上游请求，WBI 缓存后只请求搜索；
- 当前页成功渲染后最多预加载下一页 1 页，不预加载第三页，不连续扫描十页；
- 查询变化取消旧请求，旧响应不能覆盖新结果；
- 搜索响应直接包含标签、下载和删除状态。

## 3. 账号、会话与权限验收

- 每次登录创建独立 HttpOnly Cookie Token，SQLite 只保存 SHA-256 哈希；
- HTTPS 使用 `__Host-bili_session; Secure; SameSite=Lax; Path=/`；
- 每用户最多 10 个有效 Token，第 11 次登录按 `last_seen_at ASC, created_at ASC` 淘汰，且新会话不会被误撤销；
- 每会话独立 CSRF；改密轮换当前 Token/CSRF 并撤销其他会话；
- Windows 回环新安装创建受限 `admin / 123456`，Docker/NAS 使用一次性初始化令牌；
- 普通用户只能访问下载、任务、本人日志、本人 SSE 和本人设备导出；越权对象按不存在返回。

## 4. 任务和导出验收

- `owner_user_id` 只来自认证上下文；
- 普通用户无条件强制 `destination=device`、空分组和 `force=false`；
- 普通用户最多 10 个活动任务，终态保留 7 天且最多 100 条；管理员终态最多 500 条；
- 不同用户可同时导出同一 BV，内部临时目录和导出索引按用户命名空间隔离；
- 列表、详情、日志、操作、SSE、准备、下载和放弃导出均执行所有权检查；
- 管理员可按用户、状态、目标、关键词、创建/完成时间查询和分组显示。

## 6. Windows 验收

### 干净 clone

```bat
git clone https://github.com/kachekakaka/bili_workspace.git
cd bili_workspace
start.bat
verify.bat
```

CI 的 `windows-release` 作业从干净检出开始，使用仓库内 `vendor/windows/*.pack` 解压便携 Python、BBDown 和 FFmpeg，再执行源码校验、Ruff、完整 pytest 和 JavaScript 语法检查。

### 原地升级

```bat
git switch main
git pull --ff-only origin main
start.bat
verify.bat
```

升级保留 `config/`、`userdata/`、`downloads/` 和 `BBDown.data`。SQLite v2/v3 到 v4 的测试覆盖密码保留、旧任务/导出归属管理员、迁移前备份、失败回滚和外键检查。

## 7. Docker / QNAP 验收

```bash
git switch main
git pull --ff-only origin main
./docker/build-and-start.sh
```

CI 验证 Dockerfile/Compose 安全配置、`/data/config`、`/data/userdata`、`/downloads` 三目录、非 root/只读根文件系统，以及 Docker 模式旧数据库升级。正式标签触发 amd64/arm64 GHCR 镜像构建。

## 8. 数据库备份、恢复和回滚

迁移前自动生成：

```text
userdata/backups/bili_workspace-v<旧版本>-YYYYMMDD-HHMMSS.db
```

只保留最近 3 份。迁移在事务中执行，成功后运行 `PRAGMA foreign_key_check`；损坏或未来 schema 数据库不会被静默覆盖/降级。

回滚步骤：停止应用，复制当前 `userdata/`，恢复迁移前备份，切换匹配的旧标签/提交，再启动并检查用户、任务、导出、作品库和媒体。只回退代码而继续使用 schema v4 不受支持。
